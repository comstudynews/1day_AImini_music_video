## 생성형 AI로 만드는 나의 이야기 미니 뮤직비디오

# 1. 수업 개요

- 운영 방식: 50분 수업, 10분 휴식
- 수업 구성: 이론 20%, 실습 80%
- 최종 결과물: 1분 내외 분량의 미니 뮤직비디오

자료를 찾기, 핵심 내용을 정리, 가사 만들기, 음악과 이미지 생성 후 짧은 영상으로 완성. 하나의 결과물을 완성하는 과정에서 생성형 AI의 역할과 프롬프트 작성 방법을 익힌다. (각 도구 기능을 따로 배우는 것 아님)

---

# 2. 필수 결과물

① 콘텐츠 주제와 자료 요약

② 노래 기획안

③ 최종 가사

④ Suno에서 생성한 음악 1곡

⑤ 영상 장면 구성표

⑥ 장면별 이미지 4~6장

⑦ 30초~1분 분량의 미니 뮤직비디오

⑧ 사용한 주요 프롬프트

⑨ AI 결과에서 직접 수정한 내용

---

# 3. 최종 제출 파일

```
01_AI콘텐츠제작과정_학번_이름.pdf
02_AI미니뮤직비디오_학번_이름.mp4
```

Suno 음악을 저장할 수 있으면 다음 파일도 함께 제출한다.

```
03_AI생성음악_학번_이름.mp3
```

MP3 저장이 어려우면 Suno 음악 공유 주소를 PDF에 기록한다.

# 4. 시간 배분

## 4.1 오전 수업

ChatGPT와 Suno를 사용하여 다음 작업을 진행한다.

① 주제 조사와 출처 확인

② 핵심 사실, 키워드, 이야기 재료 정리

③ 1분 분량의 가사 작성

④ 음악 분위기와 스타일 설정

⑤ 영상 장면 구성 작성

⑥ Suno에서 음악 생성과 최종 곡 선택

## 4.2 오후 수업

Canva를 사용하여 다음 작업을 진행한다.

① 장면별 이미지 제작

② 이미지와 음악 배치

③ 화면 전환과 움직임 적용

④ 제목, 가사, 자막 입력

⑤ MP4 영상 출력

⑥ 제작 과정을 PDF로 정리

⑦ 제출 파일 확인

# 5. 학습환경 준비

## 5.1 준비물

- 인터넷에 연결된 노트북
- Chrome 또는 Edge 브라우저
- ChatGPT, Suno, Canva 계정 (가능하면 같은 Google 계정으로)

## 5.2 실습 폴더 만들기

컴퓨터에 다음 폴더를 만든다.

```
1일차_AI미니뮤직비디오_학번_이름
```

하위 폴더는 다음과 같이 구성한다.

```
1일차_AI미니뮤직비디오_학번_이름/
├── 01_기획자료
├── 02_음악
├── 03_이미지
└── 04_제출
```

| 폴더 | 저장 내용 |
| --- | --- |
| 01_기획자료 | 조사 결과, 출처, 가사, 장면 구성, 사용 프롬프트 |
| 02_음악 | Suno에서 생성한 음악 |
| 03_이미지 | 장면별 이미지 |
| 04_제출 | 최종 PDF, MP4, MP3 |

## 5.3 기본 작업 도구

이번 실습은 다음 세 가지 도구를 중심으로 진행한다.

```
ChatGPT → Suno → Canva → 최종 제출
```

### 5.3.1 ChatGPT

- 접속 주소: `https://chatgpt.com`
- 역할: 조사, 출처 확인, 가사 작성, 음악 스타일 설정, 장면 구성, 보고서 초안 작성

다음과 같은 새 대화를 만든다.

```
AI 미니 뮤직비디오 제작_학번_이름
```

주제 조사부터 장면 구성까지 같은 대화에서 이어서 작업한다.

### 5.3.2 Suno

- 접속 주소: `https://suno.com`
- 음악 생성 화면: `https://suno.com/create`
- 역할: ChatGPT에서 작성한 가사와 음악 스타일을 바탕으로 노래 생성
- 사용 메뉴: Create → Custom

두 곡 이상 생성한 뒤 가장 적합한 곡을 선택하고 MP3 파일이나 공유 주소를 저장한다.

### 5.3.3 Canva

- 접속 주소: `https://www.canva.com`
- 역할: 이미지 제작, 음악 삽입, 자막 작성, 영상 편집, PDF와 MP4 출력
- 사용할 유형: 동영상, 모바일 동영상, A4 문서

Canva에서 장면 이미지를 만들고 음악과 자막을 배치한 뒤 최종 영상과 제작 과정 보고서를 출력한다.

---

## 5.4 대체 도구 사용

이미지와 영상은 Gemini 또는 ChatGPT에서 제작한다. Canva는 생성한 영상 클립, Suno 음악, 가사 자막을 결합하여 최종 MP4 파일로 출력할 때만 사용한다. 수강생 계정에서 하나의 도구로 전체 편집과 출력이 가능하면 Canva 단계는 생략한다.

기본 도구에서 작업이 어려운 경우에만 다른 도구를 사용한다.

| 작업 | 기본 도구 | 대체 도구 |
| --- | --- | --- |
| 자료 조사 | ChatGPT | Perplexity, Gemini |
| 긴 문서 분석 | ChatGPT | NotebookLM |
| 이미지 생성 | Canva | ChatGPT 이미지 생성, Gemini |
| 영상 편집 | Canva | 학교에서 사용할 수 있는 영상 편집 프로그램 |
| 보고서 작성 | Canva | Microsoft Word, Google Docs |

한 프로젝트에서는 가능하면 같은 이미지 생성 도구를 계속 사용한다. 도구를 자주 바꾸면 화풍과 인물 모습이 달라질 수 있다.

---

## 5.5 전체 작업 순서

```
1. ChatGPT에서 조사, 가사, 음악 스타일, 장면 구성 완성
2. Suno에서 음악 생성과 저장
3. Canva에서 이미지 제작과 영상 편집
4. Canva에서 MP4 영상 출력
5. Canva에서 제작 과정 보고서를 PDF로 출력
6. 최종 파일을 제출 폴더에 저장
```

각 단계의 작업을 마친 뒤 다음 도구로 이동한다.

---

## 5.6 자료 입력 시 주의사항

다음 자료는 생성형 AI에 입력하지 않는다.

- 주민등록번호, 전화번호, 주소
- 비밀번호와 계정 정보
- 공개되지 않은 학교 또는 회사 자료
- 다른 사람의 개인적인 사연
- 허락받지 않은 노래 가사 전체
- 유료 이미지와 영상 원본
- 특정 가수의 목소리를 복제하려는 자료
- 사실 확인이 되지 않은 내용을 실제 사건처럼 표현한 자료

지역, 역사, 학교, 인물, 사건을 주제로 사용할 때는 날짜, 인물명, 장소명, 사건 내용을 출처와 대조하여 확인한다.

## (프로젝트 주제와 이야기 재료 준비하기)

# 6. 프로젝트 주제 선택

프로젝트는 조사형 또는 경험형 중 하나를 선택한다.

## 6.1 조사형 프로젝트

지역, 학교, 역사, 문화처럼 사실과 출처 확인이 필요한 주제에 적합하다.

### 주제 예시

- 한성대학교의 하루
- 우리 학과 소개
- 내가 사는 지역의 명소
- 지역의 역사와 문화
- 전공 분야의 미래
- 교내 행사와 축제
- 잘 알려지지 않은 지역 이야기
- 환경 보호와 지역사회 캠페인

### 작업 흐름

```
자료 조사
→ 출처 확인
→ 핵심 사실과 이야기 재료 정리
→ 노래 기획
```

ChatGPT 또는 Gemini 중 하나를 선택하여 조사부터 노래 기획까지 같은 대화에서 진행한다. 웹 검색이나 출처 확인이 어려운 경우에만 Perplexity를 사용한다.

## 6.2 경험형 프로젝트

개인의 기억, 생각, 감정이 중심인 주제에 적합하다.

### 주제 예시

- 새 학기를 시작하는 나
- 시험을 준비하는 친구
- 논문을 쓰는 대학원생
- 직장생활에 지친 동료
- 가족에게 전하는 감사
- 반려동물과의 추억
- 다시 시작하는 나
- 우리 팀 프로젝트 이야기
- 10년 후의 나
- 내가 좋아하는 장소

### 작업 흐름

```
AI와 질문하고 답하기
→ 경험과 감정 정리
→ 핵심 메시지 선택
→ 노래 기획
```

ChatGPT 또는 Gemini가 질문을 하면 자신의 경험을 답한다. 다른 사람의 사연이나 개인정보는 입력하지 않는다.

# 7. 조사형 프로젝트 수행 방법

조사형 프로젝트는 ChatGPT에서 자료 조사, 출처 확인, 이야기 재료 정리까지 이어서 진행한다.

## 7.1 자료 조사하기

ChatGPT에서 웹 검색을 사용하고 다음 프롬프트를 입력한다.

```
나는 [주제]를 바탕으로 1분 분량의 음악 콘텐츠를 만들려고 한다.

웹에서 신뢰할 수 있는 자료를 조사하여 다음 내용을 정리해 줘.

1. 주제와 관련된 핵심 사실 5개
2. 영상으로 표현하기 좋은 구체적인 장면 3개
3. 음악 콘텐츠의 중심 메시지 1개
4. 가사에 활용할 핵심 키워드 10개
5. 추가 확인이 필요한 내용
6. 참고 출처

조건:
- 공식기관, 대학, 공공기관, 신뢰할 수 있는 언론의 원문을 우선할 것
- 핵심 사실마다 근거가 되는 출처를 표시할 것
- 확인되지 않은 내용은 사실처럼 작성하지 말 것
- 사실에 근거한 장면과 창작으로 보완한 장면을 구분할 것
- 아직 노래 가사는 작성하지 말 것
- 대학생이 이해하기 쉬운 표현을 사용할 것
```

## 7.2 출처 확인하기

AI가 표시한 출처 중 중요한 자료를 직접 열어 다음 내용을 확인한다.

① 출처가 실제로 존재하는가

② 핵심 사실이 원문에 적혀 있는가

③ 날짜, 수치, 인물명, 장소명이 정확한가

④ 최근 자료가 필요한 주제는 작성일이 오래되지 않았는가

확인이 끝나면 같은 대화에 다음 프롬프트를 입력한다.

```
앞에서 조사한 핵심 사실을 출처 원문과 다시 대조해 줘.

다음 형식으로 정리해 줘.

| 핵심 사실 | 확인한 출처 | 확인 결과 | 수정할 내용 |

출처에서 확인할 수 없는 내용은 삭제하거나
'추가 확인 필요'로 표시해 줘.
```

AI의 검토 결과만 믿지 말고, 핵심 출처 3개 이상은 직접 열어 확인한다.

## 7.3 참고 문서가 있는 경우

PDF, 문서, 보고서가 있으면 현재 ChatGPT 대화에 업로드하고 다음 프롬프트를 입력한다.

```
첨부한 자료만을 바탕으로 음악 콘텐츠의 이야기 재료를 정리해 줘.

1. 자료의 핵심 주제
2. 중요한 사실 3개
3. 영상으로 표현하기 좋은 장면 3개
4. 음악 콘텐츠의 중심 메시지
5. 가사에 활용할 핵심 키워드 10개
6. 자료만으로 확인할 수 없는 내용

첨부 자료에 없는 사실은 추가하지 말고,
각 내용이 어느 자료에서 나온 것인지 표시해 줘.
```

참고 자료가 많거나 ChatGPT에서 파일을 처리하기 어려운 경우에만 NotebookLM을 사용한다.

## 7.4 조사 결과 저장하기

검증이 끝난 내용만 `01_기획자료` 폴더에 저장한다.

```
프로젝트 주제:

핵심 사실:
1.
2.
3.
4.
5.

영상 장면:
1.
2.
3.

중심 메시지:

핵심 키워드:

추가 확인이 필요한 내용:

참고 출처:
1.
2.
3.
```

출처 이름과 주소는 삭제하지 않는다.

## 7.5 기본 도구를 사용할 수 없는 경우

ChatGPT에서 웹 검색을 사용할 수 없으면 Perplexity 또는 Gemini로 자료를 조사한다.

업로드할 문서가 많거나 자료 안에서만 답을 찾아야 하는데 ChatGPT에서 처리가 어려우면 NotebookLM을 사용한다.

대체 도구를 사용하더라도 조사 결과는 다시 검토하고 출처 원문을 직접 확인한다.

## 7.6 완료 결과

조사가 끝나면 다음 내용이 준비되어야 한다.

```
- 프로젝트 주제 1개
- 검증한 핵심 사실 3개 이상
- 영상과 가사에 사용할 장면 3개
- 중심 메시지 1개
- 핵심 키워드 10개
- 확인한 출처 3개 이상
```

# 8. 경험형 프로젝트 수행 방법

경험형 프로젝트는 ChatGPT의 질문에 답하면서 노래에 사용할 이야기 재료를 정리한다.

## 8.1 경험 인터뷰 진행하기

ChatGPT에 다음 프롬프트를 입력한다.

```
너는 개인의 경험을 음악 콘텐츠로 정리하는 인터뷰 진행자다.

나는 [대상]에게 전할 [주제]의 노래를 만들려고 한다.

바로 가사를 작성하지 말고, 이야기 재료를 찾기 위한 질문을 한 번에 하나씩 해 줘.

다음 내용을 확인해 줘.

1. 이 노래를 만드는 이유
2. 실제로 있었던 경험
3. 기억에 남는 장소와 장면
4. 당시 느낀 감정
5. 상대에게 전하고 싶은 말
6. 반드시 넣고 싶은 단어
7. 사용하지 않았으면 하는 표현

필요한 질문이 끝나면 내가 답한 내용만 사용하여 다음 형식으로 정리해 줘.

- 노래의 대상
- 핵심 주제
- 주요 장면 3개
- 중심 감정
- 핵심 메시지
- 가사에 사용할 단어 10개
- 가사에서 제외할 표현

내가 말하지 않은 사건이나 감정은 추가하지 말아 줘.
```

AI가 질문하면 자신의 경험을 구체적으로 답한다. 다른 사람의 개인정보나 공개하기 어려운 내용은 입력하지 않는다.

## 8.2 인터뷰 마무리하기

이야기 재료가 충분히 모였거나 질문이 길어지면 다음과 같이 입력한다.

```
지금까지 내가 답한 내용만 사용하여 이야기 재료를 최종 정리해 줘.
```

# 10. 가사 초안 만들기

앞에서 확정한 노래 기획안을 같은 ChatGPT 대화에 붙여 넣고 다음 프롬프트를 입력한다.

```
너는 자연스럽고 따라 부르기 쉬운 한국어 가사를 쓰는 작사가다.

다음 노래 기획안을 바탕으로 2분 내외의 노래 가사 초안을 작성해 줘.

[노래 기획안 붙여넣기]

작성 조건:
- 전체 가사는 16줄에서 24줄 정도로 작성할 것
- 각 줄은 노래로 부르기 쉽도록 짧게 쓸 것
- 기획안의 주요 장면을 2개 이상 반영할 것
- 후렴에는 핵심 메시지가 분명하게 드러나는 문장을 넣을 것
- 후렴을 제외한 부분에서는 같은 뜻을 반복하지 말 것
- 조사 사실을 설명문처럼 나열하지 말 것
- 구체적인 장면과 감정이 자연스럽게 이어지게 할 것
- 쉽고 자연스러운 한국어를 사용할 것
- 지나치게 교훈적이거나 과장된 표현은 피할 것
- 제공한 자료에 없는 사실이나 경험은 추가하지 말 것
- 특정 가수의 창법, 기존 노래의 문장이나 표현을 모방하지 말 것

다음 구조로 작성해 줘.

[Verse 1]
첫 번째 장면과 상황

[Chorus]
노래의 핵심 메시지

[Verse 2]
두 번째 장면과 감정의 변화

[Bridge]
전하고 싶은 생각이나 감정의 전환

[Final Chorus]
핵심 메시지를 다시 강조하되 마지막 부분을 조금 변화시킬 것

[Outro]
짧은 마무리

가사만 출력해 줘.
```

## 10.1 가사 구조 이해

| 구분 | 역할 |
| --- | --- |
| Verse | 장면, 사건, 감정의 흐름을 보여주는 부분 |
| Chorus | 노래의 핵심 메시지를 기억하기 쉽게 전달하는 부분 |
| Bridge | 분위기나 감정을 전환하는 부분 |
| Final Chorus | 후렴을 다시 사용하되 마지막 의미를 강화하는 부분 |
| Outro | 노래를 짧게 마무리하는 부분 |

대괄호 안의 표시는 가사 구조를 구분하기 위한 안내다. 실제 음악에서는 구조와 길이가 다르게 표현될 수 있다.

## 10.2 가사 검토하기

초안을 읽고 다음 내용을 확인한다.

① 이야기의 흐름이 자연스러운가

② 주요 장면과 핵심 메시지가 드러나는가

③ 노래로 부르기 어려운 긴 문장이 없는가

④ 같은 의미가 불필요하게 반복되지 않는가

⑤ 조사 자료나 자신의 경험에 없는 내용이 추가되지 않았는가

⑥ 기존 노래와 지나치게 비슷한 문장이 없는가

수정이 필요하면 다음 프롬프트를 입력한다.

```
작성한 가사를 검토하고 다음 기준에 맞게 다듬어 줘.

- 이야기의 흐름은 유지할 것
- 긴 문장은 짧게 나눌 것
- 어색한 표현은 자연스러운 한국어로 고칠 것
- 후렴은 더 쉽고 기억하기 좋게 만들 것
- 불필요한 반복은 줄일 것
- 자료에 없는 내용은 삭제할 것
- 전체 분량은 2분 내외의 노래에 맞게 유지할 것

수정한 가사만 출력해 줘.
```

가사가 길게 느껴지면 Bridge나 Outro를 줄이고, 핵심 장면과 후렴을 남긴다.

# 12. Suno용 음악 스타일 작성

```
다음 노래 기획안과 가사에 어울리는 Suno용 음악 스타일을 작성해 줘.

노래 주제:
[주제]

핵심 감정:
[감정]

원하는 장르:
[장르]

보컬:
[남성, 여성, 혼성, 차분한 보컬, 힘 있는 보컬 등]

속도:
[느림, 중간, 빠름]

강조할 악기:
[어쿠스틱 기타, 피아노, 드럼, 신시사이저 등]

노래 기획안과 가사:
[노래 기획안과 가사 붙여넣기]

작성 조건:
- 장르, 감정, 보컬, 속도, 주요 악기를 포함할 것
- 노래의 전체 분위기와 후렴의 특징을 반영할 것
- 비슷한 뜻의 표현을 반복하지 말 것
- 특정 가수, 그룹, 기존 노래 이름을 사용하지 말 것
- Suno의 Style of Music 입력란에 바로 붙여 넣을 수 있는 한 문장으로 작성할 것
- 최종 결과는 영어로 작성할 것
- 설명은 제외하고 음악 스타일 문장만 출력할 것
```

## 12.1 결과 예시

```
Warm acoustic pop with a hopeful and reflective mood, soft female vocals, medium tempo, acoustic guitar and light piano, gentle percussion, and a clear, memorable chorus.
```

이 문장을 Suno의 `Style of Music` 입력란에 붙여 넣는다.

## 12.2 음악 스타일 구성

음악 스타일에는 다음 요소를 포함한다.

| 요소 | 작성 예시 |
| --- | --- |
| 장르 | acoustic pop, ballad, indie rock |
| 분위기 | warm, hopeful, calm, emotional |
| 보컬 | soft female vocals, warm male vocals |
| 속도 | slow tempo, medium tempo, upbeat |
| 악기 | acoustic guitar, piano, light drums |
| 곡의 특징 | memorable chorus, gentle build-up |

모든 요소를 지나치게 많이 넣으면 방향이 흐려질 수 있으므로 중요한 특징만 선택한다.

## 12.3 원하지 않는 요소 제외하기

특정 악기나 표현 방식을 제외하고 싶다면 Suno의 `Advanced Options`에서 `Exclude` 항목을 사용한다.

예:

```
heavy distortion, aggressive vocals, orchestral instruments
```

`Style of Music`에는 원하는 음악의 특징을 입력하고, `Exclude`에는 원하지 않는 요소만 입력한다.

# 13. Suno에서 음악 생성하기

## 13.1 음악 생성

① `https://suno.com/create`에 접속하여 로그인한다.

② `Create` 화면에서 `Custom` 모드를 선택한다.

③ 각 입력란에 다음 내용을 넣는다.

| 입력란 | 입력 내용 |
| --- | --- |
| Lyrics | 최종 가사 |
| Style of Music | 앞에서 작성한 영어 음악 스타일 |
| Title | 확정한 노래 제목 |

④ 가사가 없는 연주곡이 되지 않도록 `Instrumental` 옵션이 꺼져 있는지 확인한다.

⑤ `Create`를 눌러 음악을 생성한다.

⑥ 생성된 결과를 처음부터 끝까지 듣고 다음 내용을 확인한다.

- 가사와 주제가 자연스럽게 전달되는가
- 보컬의 발음이 알아듣기 쉬운가
- 분위기와 속도가 기획안에 맞는가
- 후렴이 분명하고 기억하기 쉬운가
- 영상에 사용할 수 있는 길이인가
- 음악이 갑자기 끊기거나 부자연스럽게 끝나지 않는가

⑦ 생성된 곡을 비교한 뒤 영상에 사용할 한 곡을 선택한다.

Suno가 생성하는 음악의 길이와 구성은 가사와 스타일에 따라 달라질 수 있으므로, 정확히 2분으로 생성된다고 가정하지 않는다.

## 13.2 결과 수정

수정할 곡의 더보기 메뉴에서 `Reuse Prompt`를 선택하면 기존 가사, 음악 스타일, 제목을 불러와 필요한 부분만 바꿀 수 있다.

### 속도가 너무 느린 경우

`Style of Music`에서 템포 표현을 다음과 같이 수정한다.

```
Medium tempo, brighter rhythm, light and energetic percussion
```

### 분위기가 지나치게 무거운 경우

```
Warm and hopeful mood, light arrangement, gentle acoustic instruments
```

### 보컬과 가사가 잘 들리지 않는 경우

```
Clear lead vocals, simple arrangement, sparse instrumentation, vocals forward in the mix
```

스타일을 바꿔도 가사가 잘 들리지 않으면 긴 문장을 짧게 나누고, 발음하기 어려운 표현을 자연스러운 단어로 고친다.

### 후렴이 분명하지 않은 경우

```
A clear and memorable chorus with a strong melodic hook
```

수정한 뒤 다시 생성하여 이전 결과와 비교한다. 한 번에 여러 조건을 바꾸면 원인을 판단하기 어려우므로, 가장 큰 문제부터 하나씩 수정한다.

## 13.3 음악 저장

최종 곡을 선택한 뒤 다음 중 하나의 방법으로 저장한다.

### MP3 파일로 저장

곡의 더보기 메뉴에서 다운로드 기능을 선택하여 MP3 파일로 저장한다.

파일 이름은 다음과 같이 지정한다.

```
03_AI생성음악_학번_이름.mp3
```

### 공유 주소 저장

MP3를 저장하기 어렵다면 곡의 `Copy Link` 기능으로 공유 주소를 복사한다. 복사한 주소는 제작 과정 PDF에도 기록한다.

## 13.4 작업 결과 정리

작업 결과는 다음과 같이 나누어 저장한다.

### `01_기획자료` 폴더

```
- 노래 기획안
- 가사 초안
- 직접 검토한 최종 가사
- Suno 음악 스타일
- 음악 수정 내용
- Suno 공유 주소
```

### `02_음악` 폴더

```
- 선택한 최종 음악 파일
```

최종적으로 사용할 곡의 제목, 파일 이름, 공유 주소가 서로 일치하는지 확인한다.

## (음악을 영상 장면으로 바꾸기)

# 14. 영상 길이와 장면 구성

첫날 실습 영상은 **30초에서 1분 사이**로 만든다.

음악 전체를 모두 사용할 필요는 없다. **도입부, 1절, 후렴 중에서 영상에 가장 잘 맞는 구간만 선택**한다.

영상은 보통 **4개에서 6개 장면**으로 구성한다.

장면 하나는 **약 5초에서 10초** 정도로 잡으면 무리 없이 편집할 수 있다.

장면을 정할 때는 다음 기준을 따른다.

- 가사의 흐름이 보이도록 구성할 것
- 감정 변화가 드러나도록 할 것
- 지나치게 복잡한 연기보다 이미지로 표현하기 쉬운 장면을 사용할 것
- 시작, 전개, 감정 고조, 마무리가 자연스럽게 이어지도록 할 것
- 전체 영상의 분위기와 화풍이 일관되도록 할 것
- 이미지 안에는 글자를 넣지 않을 것

---

## 14.1 사용할 구간 정하기

최종 가사를 보면서 영상에 사용할 부분을 먼저 정한다.

예를 들면 다음과 같이 정할 수 있다.

- 도입부 + 1절
- 1절 + 후렴
- 도입부 + 후렴
- 1절 일부 + 후렴 일부

핵심은 **짧은 시간 안에 주제와 감정이 가장 잘 드러나는 구간을 고르는 것**이다.

## 14.2 영상 장면표 만들기

최종 가사를 같은 ChatGPT 대화에 붙여 넣고 다음 프롬프트를 입력한다.

```
다음 가사를 바탕으로 30초에서 1분 분량의 미니 뮤직비디오 장면표를 작성해 줘.

[최종 가사 붙여넣기]

조건:
- 장면 수는 4개에서 6개로 할 것
- 각 장면은 약 5초에서 10초 사이로 구성할 것
- 노래의 흐름과 감정 변화를 반영할 것
- 이미지로 만들기 쉬운 장면을 사용할 것
- 시작, 전개, 감정 고조, 마무리가 드러나게 할 것
- 전체 분위기와 시각적 톤이 일관되게 이어지게 할 것
- 이미지 안에는 글자가 들어가지 않게 할 것
- 같은 장면을 의미 없이 반복하지 말 것
- 복잡한 카메라 움직임이나 어려운 연출은 피할 것

다음 표 형식으로 작성해 줘.

| 장면 | 시간 | 사용할 가사 | 화면에 보일 내용 | 분위기 | 핵심 이미지 요소 |
```

## 14.3 장면표 확인 기준

장면표가 나오면 다음 내용을 확인한다.

① 각 장면이 실제로 이미지나 짧은 영상으로 만들 수 있는가

② 장면 수가 너무 많거나 너무 적지 않은가

③ 감정 흐름이 자연스러운가

④ 후렴이나 핵심 메시지가 빠지지 않았는가

⑤ 장면마다 핵심 요소가 분명한가

⑥ 비슷한 화면이 반복되지 않는가

장면이 너무 많으면 줄이고, 설명이 너무 복잡하면 더 단순한 이미지 중심 장면으로 바꾼다.

## Canva에서 미니 뮤직비디오 만들기

# 16. Canva 영상 프로젝트 만들기

Canva는 앞에서 만든 이미지와 Suno 음악을 결합하고, 자막과 장면 전환을 추가하여 최종 영상을 만드는 데 사용한다.

## 16.1 영상 형식 선택

① `https://www.canva.com`에 접속하여 로그인한다.

② `디자인 만들기`에서 영상 방향에 맞는 형식을 선택한다.

| 영상 방향 | Canva 디자인 유형 | 권장 비율 |
| --- | --- | --- |
| 가로형 | 동영상 | 16:9 |
| 세로형 | 모바일 동영상 | 9:16 |

③ 빈 프로젝트를 연다.

영상 제작을 시작한 뒤 화면 비율을 바꾸면 이미지 배치가 달라질 수 있으므로, 처음에 방향을 확정한다.

## 16.2 이미지 배치

① 왼쪽 메뉴에서 `업로드`를 선택한다.

② 장면별 이미지 4장에서 6장을 업로드한다.

③ 장면표의 순서에 맞게 이미지를 한 장면씩 배치한다.

④ 이미지가 화면을 채우도록 크기를 조정하고, 중요한 인물이나 사물이 잘리지 않았는지 확인한다.

모든 장면에서 화면 비율, 화풍, 인물 모습이 자연스럽게 이어져야 한다.

## 16.3 장면 길이 조정

타임라인에서 각 장면의 길이를 조정한다.

- 장면당 약 5초에서 10초를 기준으로 시작한다.
- 가사가 바뀌거나 감정이 전환되는 지점에 맞춰 장면을 넘긴다.
- 전체 길이는 선택한 음악 구간에 맞춰 30초에서 1분으로 조정한다.

모든 장면을 같은 길이로 맞출 필요는 없다. 핵심 장면은 조금 길게, 연결 장면은 짧게 배치한다.

## 16.4 음악 추가

① Suno에서 저장한 MP3 파일을 Canva에 업로드한다.

② 음악 파일을 타임라인의 오디오 트랙에 배치한다.

③ 트랙의 양 끝을 드래그하거나 오디오 자르기 기능을 사용하여 필요한 구간만 남긴다.

④ 음악이 시작되는 지점과 첫 장면의 시작을 맞춘다.

⑤ 영상 끝에서 음악이 갑자기 끊기면 페이드아웃을 적용한다.

Canva에서는 오디오 구간 자르기, 음량 조절, 페이드인과 페이드아웃을 설정할 수 있다.

음악 파일을 저장하지 못한 경우에는 저작권 문제가 없는 수업용 음원이나 Canva에서 사용할 수 있는 오디오를 사용한다.

## 16.5 장면과 음악 맞추기

영상을 처음부터 재생하면서 음악과 장면 전환 시점이 맞는지 확인한다.

다음 부분을 기준으로 장면을 조정한다.

- 노래가 시작되는 지점
- 가사가 바뀌는 지점
- 후렴이 시작되는 지점
- 음악이 강해지거나 잔잔해지는 지점
- 마지막 메시지가 나오는 지점

Canva 계정에서 `Beat Sync` 기능을 사용할 수 있으면 음악의 박자에 맞춰 장면을 자동 조정한 뒤 결과를 다시 확인한다.

## 16.6 자막 넣기

모든 가사를 자막으로 넣지 않고, 내용을 이해하는 데 필요한 문장만 선택한다.

- 영상 제목
- 후렴의 핵심 문장
- 마지막 메시지

자막은 한 화면에 한두 줄로 작성하고, 배경과 구분되는 글자색이나 반투명 배경을 사용한다. 글자가 화면 가장자리에 붙거나 인물의 얼굴을 가리지 않도록 배치한다.

## 16.7 장면 전환 적용

페이지 사이에 하나의 부드러운 전환 효과를 선택하여 일관되게 적용한다.

권장 방식은 다음과 같다.

- 페이드
- 디졸브
- 짧고 부드러운 전환

전환 효과가 길거나 화려하면 음악과 장면에 집중하기 어렵다. 특별한 이유가 없다면 여러 종류의 전환을 섞지 않는다.

## 16.8 중간 결과 확인

영상을 처음부터 끝까지 재생하고 다음 내용을 확인한다.

① 전체 길이가 30초에서 1분 사이인가

② 이미지 순서가 장면표와 일치하는가

③ 음악과 장면 전환 시점이 자연스러운가

④ 자막을 충분히 읽을 수 있는가

⑤ 이미지가 잘리거나 빈 공간이 생기지 않았는가

⑥ 음악이 갑자기 시작하거나 끊기지 않는가

⑦ 전체 영상의 분위기와 화풍이 일관되는가

문제가 있는 장면만 수정한 뒤 다시 전체 영상을 재생한다.

- 사실과 출처를 확인한 방법

# 20. 제작 과정 PDF 저장

## 20.1 Canva에서 작성하는 경우

Canva에서 `문서` 또는 A4 문서를 만들고 ChatGPT에서 정리한 내용을 붙여 넣는다.

표지, 본문, 대표 이미지와 출처를 배치한 뒤 다음 순서로 PDF를 저장한다.

```
공유
→ 다운로드
→ 파일 형식: PDF 표준
→ 다운로드
```

파일 이름은 다음과 같이 지정한다.

```
01_AI콘텐츠제작과정_학번_이름.pdf
```

## 20.2 대체 문서 도구

Canva에서 긴 문서를 편집하기 어려운 경우에만 Microsoft Word, 한글 또는 Google Docs를 사용한다.

| 도구 | PDF 저장 방법 |
| --- | --- |
| Microsoft Word | 파일 → 다른 이름으로 저장 → PDF |
| 한글 | 파일 → PDF로 저장하기 |
| Google Docs | 파일 → 다운로드 → PDF 문서 |

PDF를 연 뒤 글자 깨짐, 이미지 누락, 페이지 잘림이 없는지 확인한다.

# 기본 도구를 사용하기 어려운 경우

기본 작업은 ChatGPT 또는 Gemini, Suno, Canva 순서로 진행한다. 해당 도구를 사용할 수 없을 때만 다음 방법으로 대체한다.

## 자료 조사나 문서 분석이 어려운 경우

ChatGPT의 웹 검색이나 파일 분석 기능을 사용할 수 없으면 Gemini를 사용한다. Gemini도 사용할 수 없다면 공식기관, 대학, 공공기관 등에서 직접 찾은 자료를 AI에 붙여 넣고 다음과 같이 요청한다.

```
다음 자료만을 바탕으로 음악 콘텐츠에 필요한 내용을 정리해 줘.

1. 핵심 사실
2. 영상으로 표현할 장면
3. 중심 감정
4. 핵심 메시지
5. 주요 키워드

자료에 없는 내용은 추가하지 말고,
확인할 수 없는 내용은 별도로 표시해 줘.

[자료 붙여넣기]
```

참고 문서가 많아 한 번에 분석하기 어려울 때만 NotebookLM을 사용한다.

## Suno를 사용할 수 없는 경우

Canva에서 사용할 수 있는 오디오나 수업에서 제공한 저작권 문제가 없는 음원을 선택하여 영상을 완성한다.

제작 과정 PDF에는 다음 내용을 기록한다.

- 최종 가사
- 곡 제목
- 음악 스타일 프롬프트
- 예상한 음악 분위기
- 대체 음원을 사용한 이유
- 실제 사용한 음원의 출처

## 이미지 생성이 어려운 경우

Canva에서 사용할 수 있는 이미지, 저작권 문제가 없는 공개 자료, 직접 촬영한 사진을 사용한다.

서로 다른 출처의 이미지를 함께 사용할 때는 색감과 화면 비율을 조정하여 영상 전체의 분위기를 맞춘다.

## Canva를 사용할 수 없는 경우

이미지, 음악, 자막을 결합하고 MP4로 저장할 수 있는 학교 제공 프로그램이나 사용 가능한 영상 편집 도구를 이용한다.

대체 도구를 사용하더라도 다음 조건은 유지한다.

- 영상 길이: 30초에서 1분
- 장면 수: 4개에서 6개
- 음악과 장면 순서 일치
- 핵심 자막 포함
- MP4 형식으로 저장

# 스스로 해보기

수업에서 제시한 주제와 결과물을 그대로 따라 하지 않고, 자신의 경험이나 관심 분야를 바탕으로 프로젝트를 완성한다.

```
주제 선택
→ 자료 조사 또는 경험 정리
→ 핵심 사실과 장면 선택
→ 노래 기획안 작성
→ 가사 작성
→ 가사 검토와 수정
→ 음악 생성
→ 영상 장면표 작성
→ 이미지 준비
→ 음악과 이미지 편집
→ 영상 검토
→ 제작 과정과 결과물 제출
```

AI가 만든 결과를 그대로 사용하지 않는다. 프로젝트 목적에 맞지 않는 내용은 삭제하고, 필요한 부분은 직접 수정한다.

제작 과정 PDF에는 다음 내용을 반드시 남긴다.

- AI의 제안 중 선택한 내용
- 삭제하거나 변경한 내용
- 직접 추가한 내용
- 사실과 출처를 확인한 방법
- 최종 결과를 선택한 이유
